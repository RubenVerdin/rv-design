## Getting Started

```bash
npm install rv-design
```

**Register all components as a Vue plugin:**

```ts
import { createApp } from 'vue'
import RvDesign from 'rv-design'
import 'rv-design/styles.css'

const app = createApp(App)
app.use(RvDesign)
app.mount('#app')
```

**Or import components individually:**

```ts
import { RvButton, RvInput } from 'rv-design'
import 'rv-design/styles.css'
```

## Theming

RV Design supports three theme modes driven by the `data-theme` attribute on the root element:

| Mode | Behaviour |
|---|---|
| *(not set)* | Follows the OS via `@media (prefers-color-scheme)` |
| `data-theme="light"` | Pins the light palette |
| `data-theme="dark"` | Pins the dark palette |

Use the `useTheme()` composable to manage this at runtime:

```ts
import { useTheme } from 'rv-design'

const { mode, setTheme, cycle } = useTheme()
// mode.value → 'system' | 'light' | 'dark'
setTheme('dark')
cycle() // system → light → dark → system
```

> Use the **theme toggle** in the Storybook toolbar (sun / moon icon) to preview both modes on any story.

## Design Tokens

All colours, spacing, radii, shadows, and motion values are CSS custom properties defined in `tokens/`. Components reference only semantic aliases, making full re-theming possible by overriding those aliases.

| File | Contents |
|---|---|
| `tokens/colors.css` | Raw oklch palette + semantic surface/text/border aliases |
| `tokens/typography.css` | Type scale, weights, line-heights |
| `tokens/spacing.css` | 4 px base grid (`--rv-space-1` … `--rv-space-12`) |
| `tokens/radius.css` | Border-radius tokens |
| `tokens/elevation.css` | Shadow tokens |
| `tokens/motion.css` | Easing curves and durations |
